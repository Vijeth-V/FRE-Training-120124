git clone [projectPath]: Creates a local copy of a remote repository.

git pull: Fetches changes from the remote repository and merges them into the current branch.

git checkout [branchName]: Switches to the specified branch.

git branch [branchName]: Create a new branch called branchName.

git checkout -b [branchName]: Creates a new branch and switches to it.

git branch: Lists all the existing branches.

git add .: Stages all changes in the current directory for the next commit.

git commit: Saves the staged changes as a new commit.

git log: Shows the commit history of the repository.

git push: Uploads local branch changes to the remote repository.

git merge: Combines changes from one branch into another.

git reset vs git revert: git reset is for undoing uncommitted changes, while git revert as a tool for undoing committed changes.

git merge vs git rebase: Merging combines branches by preserving commit history, while rebasing rewrites history by applying commits linearly.

git pull vs git fetch: git pull fetches and merges changes, while git fetch only retrieves changes without merging.
